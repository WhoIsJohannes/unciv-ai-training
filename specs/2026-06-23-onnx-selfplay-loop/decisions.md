## Default-rigor design decisions (Claude, technical)
- **D1 Terminal reward + VERSION bump**: shards record no reward/terminal today (hardcoded 0). Add per-civ terminal ±1 at game end; bump `SampleSchema.VERSION` 1→2; lockstep `python schema.py SCHEMA_VERSION=2`. Reader refuses old shards → regenerate each round (perishable-dataset design). Sanctioned by the task's NON-GOAL escape hatch.
- **D2 RoutingPolicy**: injection is one-policy-per-run; wrap with `RoutingPolicy(learnerId, onnx, random)` that routes by civ — no data-plane modification. Pin fixed nations in the Tiny config so learner identity (and `numWins[learner]`) is stable across games.
- **D3 SimStats extraction**: extract `binomialTest`/`normalCdf`/`erf` from `Simulation` into a public `SimStats` util for EVAL reuse + unit-testing; EVAL prints one machine-readable result line.
- **D4 onnxruntime placement** (REVISED after Step 5 council FND-0002/0006 + Step 7 scan): **OnnxPolicy lives in `desktop`** (where the ORT native dep belongs), implementing `core`'s `PolicyProvider`. `RoutingPolicy` + `SimStats` (no ORT dep) stay in `core/.../dataplane` + `core/.../simulation`. onnxruntime = a `desktop`-only `implementation` dep (added to `gradle/libs.versions.toml`). This keeps `core` ML-runtime-free (no dependency inversion, no compileOnly hack, zero Android risk) — the data plane already places heavy concerns (DataPlaneGen) in desktop. **Deviation from the task's "OnnxPolicy in core/.../dataplane" — justified; surfaced in the plan Deviations table.**
- **D6 Per-(civ,turn) inference cache** (council FND-0037): `chooseIndex` is called once per head (tech, then policy) per civ-turn → OnnxPolicy runs ONE forward pass per (civID, turn), memoizing both heads' logits; the 2nd head reuses it. Memo holds only the current (civ,turn) (tiny, cleared on key change).
- **D7 Win/tiebreaker (precise)** (council FND-0023/0027/0009): win = formal victory; else (only at the hard turn cap) the civ with higher `calculateTotalScore()`; exact tie = draw (reward 0, excluded from the win numerator). SAME for training reward + EVAL. Turn cap defaults high so the tiebreaker is the rare path (user decision).
- **D8 Lean surface** (council FND-0001/0003/0005/0004): ~6 driver knobs (K, gen-games, eval-games, turn-cap, threads, seed) with baked balanced defaults (the default operating contract); plain-script driver (no framework); hardcode the 2 heads (no generic head abstraction); `SimStats` = just the binomial test.
- **D9 Error handling + native memory** (council FND-0007/0022/0025): close `OnnxTensor`/`OrtSession.Result`; one shared session closed at run end; model-load/inference failure → fail fast (loud), never silent-fallback; model path is operator-supplied + self-produced (trusted).
- **D10 JVM↔Python contract** (council FND-0008/0021): driver invokes `./gradlew selfPlayGen|selfPlayEval` with explicit args; EVAL prints exactly one line `EVAL_RESULT <json>` (games, wins, winrate, pval, learner, seed, onnx_decisions) the driver parses; routing-held assertion (`onnx_decisions>0`).
- **D11 Documented v1 limitations** (council FND-0030/0033/0029): fixed-nation matchup → matchup-overfitting (acceptable for a stationary-opponent go/no-go signal; map seed varies per game; v2 = randomize nations); undiscounted return → high variance (baseline mitigates; γ optional knob, default 1.0); no opponent co-learning (v1 = learner-vs-Random, per task non-goal).
- **D12 Loop hygiene** (council FND-0011/0039/0013): turn cap bounds every game (no infinite hang) + a generous per-process timeout backstop in the driver; fresh per-round shard dir with retention (keep last N, configurable); VERSION-bump rollback = bump back + regen (perishable-by-design).
- **D13 Parity tolerance** (council FND-0024): PARITY test asserts logits match within `atol=1e-4` (f32).
- **D5 ONNX I/O contract**: input = concat(`global`,`acting_civ`) at the runtime GnK width; outputs `tech_logits[techCount]`, `policy_logits[policyCount]`. Widths read from vocab/schema (NOT hardcoded 80/70). VERSION + ruleset fingerprint stamped as ONNX metadata + a shared constant. PARITY test = JVM session vs Python reference on a fixed observation.

## Plan-council decisions (Step 11, from 42 findings)
- **D14 CONTROL wiring (THE fix, new P0)**: the installed policy must DRIVE tech+policy for the civs it controls, not just record labels (`onCivTurn` is recording-only today; the heuristic `chooseTechToResearch`/`adoptPolicy` make the real decisions). Design: `DataPlaneHooks` splits per-civ-turn handling into **controlApply** (always — choose+apply) and **emit** (gen only). Decision-gated: record/apply a TECH action only when `civInfo.tech.techsToResearch.isEmpty()` (pre-fill the queue → `chooseTechToResearch` respects it, line 276); record/apply a POLICY action only when `civInfo.policies.canAdoptPolicy()` (adopt the chosen policy); else record −1 (no decision this turn). `chooseIndex` is called ONCE per head per decision-turn → recorded action == applied action. Guard `NextTurnAutomation.adoptPolicy` to skip the heuristic loop for controlled civs. Control runs in BOTH gen + eval (eval installs the policy without registering a recorder). greatPerson/diplomaticVote stay heuristic (recorded labels only; not trained). Symmetric: learner = net for tech/policy + heuristic rest; opponent = random for tech/policy + heuristic rest → the learning signal is isolated to tech+policy.
- **D15 ThreadLocal memo**: OnnxPolicy's per-(civ,turn) forward-pass cache is a `ThreadLocal` keyed by (gameId, civID, turn) holding only the most recent — fixes concurrent-game corruption (FND-0001/0026/0023).
- **D16 Loss correctness**: per-head masking — skip a head's term when its recorded action `< 0`; assert `a>=0` before gather (no negative-index bug); keep a step if ANY head acted (don't drop whole steps) (FND-0024/0025).
- **D17 Go/no-go rule**: GO if final-round winrate ≥60% AND p<0.05; PLATEAU if winrate ∈ [45,55]% with no upward trend over the last ≥3 rounds; INCONCLUSIVE otherwise. The driver emits a verdict line alongside curve.csv (FND-0010/0018).
- **D18 Parity fixture**: `selfPlay parity-dump <seed>` writes a committed fixture observation; JVM-parity and Python-parity both read the SAME fixture; assert logits within 1e-4 (FND-0015).
- **D19 Driver/runtime robustness**: subprocess as an **arg-list** (no shell, no string interpolation); check returncode + per-call timeout; a failed round aborts loudly. OnnxPolicy inference error → abort THAT game (isolation), not the whole run. Validate model path exists (FND-0005/0006/0017/0019/0020/0021). Local-trust context: driver + JVM are the same trusted user.
- **D20 Concrete defaults + budget**: K=10, gen-games=24, eval-games=100 (60% vs 50% ⇒ p≈0.023), turn-cap=325 (high, per the win-rule decision), threads=cores−1, ORT intra-op=1. gen→train→eval are SEQUENTIAL phases ⇒ no JVM/ORT/torch thread contention. Hard total-games budget cap. Shard retention (keep-last-N) is P0 (per-round regen fills disk). dataset groups by (shard, civ_slot); training sorts shards by name (FND-0011/0033/0036/0007/0035/0027/0004/0034/0039).
- **D21 Responsible-use note (not engineered)**: a tech/policy picker for a turn-based strategy game has negligible real-world harm/dual-use; plan adds a one-paragraph scope note, no harm-reporting machinery (FND-0028/0029/0030/0031/0032 — over-reach for a game AI).
